K-Nearest Neighbors (KNN)

The K-Nearest Neighbors (KNN) classifier is a simple, instance-based learning algorithm used for classification tasks. It works by classifying a new data point based on the majority class of its nearest neighbors in the feature space.

Pros:

  • Simple and Easy to Understand: KNN is intuitive and easy to implement.
  • No Training Phase: KNN doesn't explicitly "train" a model; instead, it stores the training dataset and uses it for classification during prediction, making it a lazy learner.
  • Effective for Small Datasets: Works well on smaller datasets with lower dimensionality.
  • Adaptable to Multi-Class Classification: It can handle both binary and multi-class classification tasks.

Cons:

  • Computationally Expensive for Large Datasets: Since KNN doesn't generalize from the data, it needs to search through the entire training set during prediction, which can be slow for large datasets.
  • Sensitive to Noisy Data: KNN may be sensitive to outliers or irrelevant features because it doesn't generalize the data. Feature scaling is often required.
  • Requires Good Distance Metric: The choice of distance metric (e.g., Euclidean, Manhattan) and the number of neighbors (k) can significantly affect performance.
  • Curse of Dimensionality: Performance deteriorates with increasing dimensionality because the concept of "closeness" becomes less meaningful in high-dimensional spaces.

# data preparaion
dataset = load_iris()
data = dataset.data
target = dataset.target

cols = dataset.feature_names
cols = [x[:-5].replace(' ','_') for x in cols]
df = pd.DataFrame(data=np.c_[data,target], columns=cols+['target'])
df.drop_duplicates(inplace=True)

X = df.copy()
y = X.pop('target')

scaler = StandardScaler()
scaler.fit(X)
X = scaler.transform(X)
X_train, X_test, y_train, y_test = train_test_split(X,y,test_size=0.2)
# Only the first two features (sepal length and sepal width) are used for simplicity in visualizing decision boundaries.
X_train = X_train[:, :2]
X_test = X_test[:, :2]

Explanation of Hyperparameters:

  • n_neighbors: Determines the number of nearest neighbors to use for classification.

  • weights: Specifies whether all neighbors contribute equally (uniform) or closer ones have more influence (distance).

  • algorithm: Defines the method used to compute nearest neighbors (auto, ball_tree, kd_tree, or brute).

  • leaf_size: Controls the number of points stored in the leaf nodes of the tree, affecting query speed and memory usage.

  • p: Chooses the distance metric: p=1 for Manhattan distance (L1) or p=2 for Euclidean distance (L2).

# K-Neighbours classifier
kneighbours = KNeighborsClassifier()
params = {
    'n_neighbors': [5, 7, 10],  
    'weights': ['uniform', 'distance'],  
    'algorithm': ['auto', 'ball_tree', 'kd_tree', 'brute'], 
    'leaf_size': [20, 30, 40],  
    'p': [1, 2]  
}

grid_search = GridSearchCV(kneighbours, params, cv=5, scoring='r2', n_jobs=-1, verbose=2)

grid_search.fit(X_train, y_train)

best_params = grid_search.best_params_
best_model = grid_search.best_estimator_

r2 = best_model.score(X_test, y_test)
print('K-Neighbours R2: {0:.2f}'.format(r2))

Plot decision Boundary

This plot visualizes how the classifier has split the feature space into two regions for the two classes.

  • Background Colors: The plot is divided into regions, each representing the classifier's prediction for that region. These regions are separated by the decision boundary, which is the line that separates the classes.

  • Points: The points plotted over the background represent the training data. They are colored according to their true class, showing where they fall in the feature space.

  • Points correctly classified by the model should be in the region corresponding to their class.

  • Misclassified points will appear in a region that is colored differently from their true class.

  • The decision boundary helps us understand how the model makes predictions and where it struggles.

Interpretation:

  • If points are mostly on the correct side of the boundary, the model is performing well.
  • If many points are on the wrong side of the boundary, this suggests that the classifier is misclassifying them.

def plot_decision_boundary(X, y, model, title="Decision Boundary"):
    x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.01),
                         np.arange(y_min, y_max, 0.01))
    
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    
    # Corrected colors using shorthand notation
    plt.contourf(xx, yy, Z, alpha=0.4)
    plt.scatter(X[:, 0], X[:, 1], c=y, s=20, edgecolor='k')
    plt.title(title)
    plt.show()

Confusion Matrix:

A confusion matrix is a table that helps visualize the performance of a classification model. The rows represent the actual classes, and the columns represent the predicted classes. A confusion matrix for 3 classes helps in visualizing how well the model is classifying multiple classes. The diagonal values represent correct predictions, while the off-diagonal values represent misclassifications.
