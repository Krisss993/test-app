Support Vector Classifier (SVC)

The Support Vector Classifier (SVC) is part of the Support Vector Machine (SVM) family, which works by finding the optimal hyperplane that separates the data into different classes with maximum margin.

Strengths:

  • Effective in high-dimensional spaces: It works well when the number of features is greater than the number of samples.
  • Versatile kernels: You can use different kernel functions (linear, polynomial, radial basis function) to handle different types of data (linearly and non-linearly separable data).
  • Regularization: The C parameter controls the trade-off between maximizing the margin and minimizing classification errors, helping avoid overfitting. Weaknesses:
  • Sensitive to noise: SVMs can be sensitive to outliers and noisy data, especially with a small value of C.
  • Computational cost: For large datasets, training can be time-consuming, especially when using non-linear kernels.
  • Hard to interpret: The decision boundary is often difficult to interpret, especially with non-linear kernels. Conclusions: SVC is a powerful classifier, especially for complex datasets with high dimensionality, but it can be resource-intensive for larger datasets. It's ideal when interpretability isn't the primary concern, and you're looking for high accuracy.

df = pd.DataFrame(data=np.c_[data,target], columns=cols+['target'])
df.drop_duplicates(inplace=True)

X = df.copy()
y = X.pop('target')

scaler = StandardScaler()
scaler.fit(X)
X = scaler.transform(X)
X_train, X_test, y_train, y_test = train_test_split(X,y,test_size=0.2)
# Only the first two features (sepal length and sepal width) are used for simplicity in visualizing decision boundaries.
X_train = X_train[:, :2]
X_test = X_test[:, :2]

Explanation of Hyperparameters:

  • C: Controls the trade-off between maximizing the margin and minimizing classification errors; smaller values create a wider margin but allow more errors.

  • gamma: Defines how far the influence of a single training point reaches, with low values meaning 'far' and high values meaning 'close'; affects the shape of the decision boundary.

  • kernel: Specifies the kernel type to use for mapping input data to a higher-dimensional space (e.g., linear, rbf for radial basis function, sigmoid).

  • degree: Used with the poly kernel, it specifies the degree of the polynomial kernel function (ignored for other kernel types).

  • coef0: Controls the influence of higher-order terms in polynomial or sigmoid kernels; useful for controlling the decision boundary's shape.

# Support Vector Classifier
svc = SVC()

params = {
    'C': [0.1, 1, 10, 100],
    'gamma': ['scale', 'auto', 0.001, 0.01, 0.1, 1], 
    'kernel': ['linear', 'rbf', 'sigmoid'], 
    'degree': [2, 3, 4],  
    'coef0': [0, 0.1, 0.5, 1]  
}

grid_search = GridSearchCV(svc, params, cv=5, scoring='accuracy', n_jobs=-1, verbose=2)

grid_search.fit(X_train, y_train)

best_params = grid_search.best_params_
best_model = grid_search.best_estimator_

print("Best Parameters:", best_params)
print("Best Cross-Validation Accuracy:", grid_search.best_score_)

r2 = best_model.score(X_test, y_test)
print("SVC R2:", r2)

Plot decision Boundary

This plot visualizes how the SVC classifier has split the feature space into two regions for the two classes.

  • Background Colors: The plot is divided into regions, each representing the classifier's prediction for that region. These regions are separated by the decision boundary, which is the line that separates the classes.

  • Points: The points plotted over the background represent the training data. They are colored according to their true class, showing where they fall in the feature space.

  • Points correctly classified by the model should be in the region corresponding to their class.

  • Misclassified points will appear in a region that is colored differently from their true class.

  • The decision boundary helps us understand how the model makes predictions and where it struggles.

Interpretation:

  • If points are mostly on the correct side of the boundary, the model is performing well.
  • If many points are on the wrong side of the boundary, this suggests that the classifier is misclassifying them.

# Decision Boundary Plot
def plot_decision_boundary(X, y, model, title="Decision Boundary"):
    x_min, x_max = X[:, 0].min() - 1, X[:, 0].max() + 1
    y_min, y_max = X[:, 1].min() - 1, X[:, 1].max() + 1
    xx, yy = np.meshgrid(np.arange(x_min, x_max, 0.01),
                         np.arange(y_min, y_max, 0.01))
    
    Z = model.predict(np.c_[xx.ravel(), yy.ravel()])
    Z = Z.reshape(xx.shape)
    
    plt.contourf(xx, yy, Z, alpha=0.4, cmap=plt.cm.RdYlBu)
    plt.scatter(X[:, 0], X[:, 1], c=y, s=20, edgecolor='k', cmap=plt.cm.RdYlBu)
    plt.title(title)
    plt.show()

Confusion Matrix:

A confusion matrix is a table that helps visualize the performance of a classification model. The rows represent the actual classes, and the columns represent the predicted classes. A confusion matrix for 3 classes helps in visualizing how well the model is classifying multiple classes. The diagonal values represent correct predictions, while the off-diagonal values represent misclassifications.
